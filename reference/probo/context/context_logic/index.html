DynamicData dataclass

data class to define and get dynamic data (from user or db) with custom manipulation plug in function for rendering components

data property

Safely accesses the full dynamic data dictionary.

Returns:
  • Dict( Dict ) –

    The dynamic data dictionary, ensuring a dict type is returned even if empty.

__post_init__()

This runs AUTOMATICALLY immediately after init. It processes the data_obj and populates dynamic_data instantly.

get(Value)

Retrieves a value from the processed dynamic data.

Parameters:
  • Value (str) –

    The key to look up.
Returns:
  • Any –

    The value associated with the key, or None if not found.

TemplateComponentMap

Template Components Map is the cordinator that serves the right component to spesific view

get_component(url_name)

Retrieves a component associated with a URL name and injects request properties.

Parameters:
  • url_name (str) –

    The key/name representing the URL route.
Returns:
  • str | tuple –

    str | tuple: The rendered component (usually a string or HTML tuple).
Raises:
  • ValueError –

    If the url_name does not exist in the map.

set_component(url_name=None, component_name=None, **url_name_comp)

Registers new components or updates existing mappings.

Parameters:
  • url_name (str, default: None ) –

    The specific URL key to update.

  • component_name (object, default: None ) –

    The component object to associate.

  • **url_name_comp –

    Additional mappings passed as keyword arguments.
Returns:
  • TemplateComponentMap –

    Self, allowing for method chaining.

TemplateProcessor

A class to process only 'mui' style template blocks but with support for generating Django-style template tags. Static methods are included.

__init__(data_context=None)

Initializes the processor with a global data context.

Parameters:
  • data_context (dict, default: None ) –

    A dictionary of global variables available to all templates rendered by this instance.

for_loop(expression, for_block, style='mui', empty_content=None) staticmethod

Generates a for-loop block string in the specified style.

Parameters:
  • expression (str) –

    The loop expression (e.g., "item in items").

  • for_block (str) –

    The content to repeat inside the loop.

  • style (str, default: 'mui' ) –

    The syntax style to generate ("mui" or "django").

  • empty_content (str, default: None ) –

    Content to display if the iterable is empty (Django style only).
Returns:
  • str –

    The formatted loop block string.

if_true(expression, if_block, style='mui', else_statement=None, **elif_statements) staticmethod

Generates a conditional block string in the specified style (MUI or Django).

Parameters:
  • expression (str) –

    The condition to evaluate (e.g., "user.is_admin").

  • if_block (str) –

    Content to render if true.

  • style (str, default: 'mui' ) –

    The syntax style to generate ("mui" or "django").

  • else_statement (str, default: None ) –

    Content for the else block.

  • **elif_statements –

    Key-value pairs where key is condition and value is content for elif blocks.
Returns:
  • str –

    The formatted conditional block string.

render_template(template_string, context=None)

The main entry point for rendering a template string.

Merges the instance's global context with the provided local context and processes all template blocks (loops, ifs, vars).

Parameters:
  • template_string (str) –

    The raw template string to render.

  • context (dict, default: None ) –

    Additional local context for this specific render.
Returns:
  • str( str ) –

    The fully rendered string.

set_variable(expression) staticmethod

Generates a variable output string.

Parameters:
  • expression (str) –

    The variable expression to output (e.g., "user.name").
Returns:
  • str –

    The formatted variable string (e.g., "{{ user.name }}").

loop(data, renderer)

Iterates over data and generates elements using a renderer function.

Parameters:
  • data –

    An int (for simple duplication), a dict, or an iterable (list/queryset).

  • renderer –

    A function/lambda that takes the item(s) and returns an Element, OR a static Element to simply duplicate.
Returns:
  • –

    A list of rendered elements.
