    shortcuts

    probo.shortcuts.shortcuts

    component(config)

    The master factory function for instantiating ProboUI Components.

    This function acts as the primary 'compiler' in the ProboUI pipeline. It consumes a ComponentConfig blueprint and performs the following orchestration: 1. Initializes a Component instance with the provided identity (name). 2. Hydrates the component's internal tree using the StateConfig. 3. Applies visual rules defined in the StyleConfig. 4. Registers children and incoming properties for rendering.

    Parameters:

    Name Type Description Default
    config ComponentConfig

    The comprehensive configuration object containing the template, state, style, and child elements.

    		 required

    Returns:

    Name Type Description
    Component Component

    A live component object ready to be rendered to an HTML string or manipulated by the framework.

    custom(tag, content='', is_void_element=False, EL=None, **attrs)

    Instantly creates and renders a custom HTML tag string.

    This utility serves as a functional shortcut for the Element factory, bypassing the need for manual instantiation and method chaining. It is ideal for generating simple, non-stateful HTML fragments or for use within list comprehensions where immediate string output is required.

    Parameters:

    Name Type Description Default
    tag str

    The HTML tag name (e.g., 'section', 'my-web-component').

    		 required
    content str

    The inner HTML or text content for the element.

    		 ''
    is_void_element bool

    If True, renders the tag as a self-closing element (e.g., ).

    		 False
    **attrs dict[str, Any]

    Arbitrary keyword arguments to be converted into HTML attributes (e.g., class_='active', id='main').

    		 {}

    Returns:

    Name Type Description
    str str

    The fully rendered HTML string.

    datatable(config)

    Compiles a TableConfig blueprint into a fully formatted HTML table string.

    This utility serves as the execution engine for data grids. It iterates through the columns to build the table header, then processes the data list to generate rows. It also handles the injection of row-level actions (like buttons or links) and applies framework-specific CSS classes.

    Parameters:

    Name Type Description Default
    config TableConfig

    The configuration blueprint containing column definitions, raw data, CSS classes, and action components.

    		 required

    Returns:

    Name Type Description
    str str

    A fully rendered HTML

    string ready for inclusion in a component or page.

    document(config)

    The master orchestrator that compiles a PageConfig into a full HTML Template.

    This function represents the final stage of the ProboUI compilation pipeline. It integrates all sub-configurations (Head, Theme, Layout, and Components) into a cohesive Template instance. It is the primary entry point for generating complete, production-ready web documents.

    Parameters:

    Name Type Description Default
    config PageConfig

    The root configuration object containing the entire state and structure of the page.

    		 required

    Returns:

    Name Type Description
    Template Template

    A fully hydrated document engine ready to be stringified into the final HTML response.

    form_field(tag, **kwargs)

    A functional factory for creating ProboFormField instances.

    This utility simplifies the creation of form field configurations by wrapping the ProboFormField constructor. It allows for a declarative definition of HTML inputs, textareas, and selects while mapping arbitrary keyword arguments directly to field attributes.

    Parameters:

    Name Type Description Default
    tag str

    The HTML tag name or type of the field (e.g., 'input', 'textarea', 'select').

    		 required
    **kwargs dict[str, Any]

    Additional attributes and configurations for the field, such as type, name, label, value, or class_.

    		 {}

    Returns:

    Name Type Description
    ProboFormField ProboFormField

    A structural blueprint of the form field ready to be processed by a FormHandler or Component.

    head_seo(config)

    Compiles a HeadConfig blueprint into a functional Head object.

    This factory function serves as the central hub for document-level metadata. It processes the configuration to initialize a Head instance that manages: 1. SEO essentials (Title, Description, Keywords). 2. Social Graph tags (OpenGraph, Twitter Cards). 3. Resource linking (Stylesheets, Scripts, Favicons). 4. Character encoding and Viewport settings.

    Parameters:

    Name Type Description Default
    config HeadConfig

    The declarative blueprint containing all metadata and external asset references.

    		 required

    Returns:

    Name Type Description
    Head Head

    A live Head object capable of stringifying into a valid HTML block.

    iterator(config)

    Compiles a ListConfig into a single, wrapped HTML string.

    This utility serves as the execution engine for repeated elements. It iterates through the provided data source, applies the item_renderer to each item, and assembles the results into a structured HTML fragment defined by the configuration's wrapper tag and attributes.

    Parameters:

    Name Type Description Default
    config ListConfig

    The configuration blueprint containing the items, the rendering function, and the wrapper metadata.

    		 required

    Returns:

    Name Type Description
    str str

    A fully rendered HTML string representing the entire collection.

    layout(config)

    Transforms a layout configuration into a renderable Template engine.

    This function acts as the high-level orchestrator for page structure. It consumes a LayoutConfig to define the structural boundaries (slots) of a page and returns a Template instance. This instance is responsible for injecting specific component content into the defined layout regions during the final render pass.

    Parameters:

    Name Type Description Default
    config LayoutConfig

    The structural blueprint defining wrapper tags, attribute metadata, and named content slots.

    		 required

    Returns:

    Name Type Description
    Template Template

    A document-level object prepared to receive body content and global metadata for full HTML generation.

    probo_form(config)

    Compiles a FormConfig into a functional ProboForm instance.

    This factory function serves as the constructor for the framework's form handling system. It processes the configuration to determine the source of truth for the fields (Django vs. Manual) and initializes a ProboForm object capable of rendering itself, handling CSRF protection, and interfacing with HTMX for partial updates.

    Parameters:

    Name Type Description Default
    config FormConfig

    The configuration object containing field definitions, action/method metadata, and optional Django integration hooks.

    		 required

    Returns:

    Name Type Description
    ProboForm ProboForm

    A live form object ready to be rendered as HTML or passed to a FormHandler for server-side processing.

    semantic_layout(config)

    Orchestrates a Template instance based on HTML5 semantic regions.

    This utility functions as a specialized compiler for structural layouts. It maps the explicit regions defined in a SemanticLayoutConfig (header, footer, aside, sections) into a Template object. This ensures that the final document follows a logical outline, improving accessibility and search engine visibility.

    Parameters:

    Name Type Description Default
    config SemanticLayoutConfig

    		 required

    Returns:

    Name Type Description
    Template Template

    A document-level engine initialized with semantic regions, ready to accept page-specific content.
