    Animation

    A high-level manager for creating and rendering CSS @keyframes.

    The Animation class provides a simplified interface for building complex motion sequences. It supports standard 'from-to' transitions, discrete percentage-based frames, and sequential frame addition.

    Attributes:

    Name Type Description
    name str

    The identifier for the animation sequence.
    frames dict

    An internal mapping of timeline steps to CSS properties.

    add_frame(step, **properties)

    Appends a single frame to the animation timeline.

    Parameters:

    Name Type Description Default
    step str

    The timeline position (e.g., '0%', '50%', 'to').

    		 required
    **properties dict[str, Any]

    CSS property-value pairs for this frame.

    		 {}

    animate_from_to(from_props, to_props)

    Quickly defines a two-point transition.

    Parameters:

    Name Type Description Default
    from_props dict

    CSS properties for the starting state ('from').

    		 required
    to_props dict

    CSS properties for the ending state ('to').

    		 required

    @keyframes myAnimation { from {background-color: red;} to {background-color: yellow;} }

    animate_percent(blocks)

    Populates the timeline using a dictionary of percentage steps.

    Parameters:

    Name Type Description Default
    blocks dict[str, dict]

    A mapping of steps to property dicts. e.g., {"0%": {"opacity": 0}, "100%": {"opacity": 1}}

    		 required

    @keyframes myAnimation { 0% {background-color: red;} 25% {background-color: yellow;} 50% {background-color: blue;} 100% {background-color: green;} }

    render()

    Serializes the frames into a valid CSS @keyframes block.

    Name Type Description
    str str

    The fully rendered @keyframes string.

    CssAnimatable

    A factory for generating validated CSS Keyframe animations.

    Attributes:

    Name Type Description
    name str

    The identifier for the animation (e.g., 'fade-in').
    animations list

    A collection of stored animation definitions.
    validator CssRuleValidator

    Parameters:

    Name Type Description Default

    		 required

    Raises:

    Type Description

    animate(name, steps=None, **properties)

    Constructs a full @keyframes string block.

    Can generate either a simple 'from' animation or a complex multi-step sequence (0%, 50%, 100%).

    Parameters:

    Name Type Description Default
    name str

    The name of the keyframe sequence.

    		 required
    steps dict

    A dictionary where keys are steps (e.g., '0%', '100%') and values are property dictionaries.

    		 None
    **properties dict[str, str]

    Keyword arguments for a single-step animation (defaults to a 'from' block).

    		 {}

    Returns:

    Name Type Description
    str str

    The fully formatted @keyframes CSS string.
    Example

    anim.animate('slide', steps={'0%': {'left': '0px'}, '100%': {'left': '100px'}}) anim.animate('fade', opacity='0')

    CssRuleValidator

    A silent validation engine for CSS property-value pairs.

    This class leverages cssutils to verify that CSS declarations follow W3C standards. It is designed to be 'silent,' meaning it suppresses library-level logging to maintain performance and terminal clarity during high-volume rendering (e.g., 1,000,000+ nodes).

    Attributes:

    Name Type Description
    property_value Dict[str, Any]

    A dictionary of CSS properties and their values to be validated.

    is_valid()

    Aggregates validation results for all properties in the instance.

    Returns:

    Name Type Description
    bool bool

    True if every property-value pair is syntactically valid CSS.

    validate_css(css_string)

    Performs a low-level syntax check on a single CSS declaration string.

    This method captures internal cssutils errors via a local logging handler to prevent terminal pollution. It checks both for hard syntax errors and for 'dropped' properties (where the parser doesn't recognize the property name).

    Parameters:

    Name Type Description Default
    css_string str

    A full CSS declaration (e.g., "color:red;").

    		 required

    Returns:

    Name Type Description
    bool bool

    True if the CSS string is valid and recognized by the parser.

    MediaQueries

    This class provides a programmatic way to define responsive design breakpoints. It validates media types (screen, print, etc.) and features (width, orientation, etc.) to ensure the generated CSS is standard-compliant.

    is_not bool

    is_only bool

    If True, prepends the 'only' operator.

    css_comment(css, return_type=str())

    Wraps a CSS string or list of rules in a CSS comment block.

    This function is primarily used for debugging, documenting rendered output, or temporarily disabling style rules within the ProboUI rendering pipeline.

    Parameters:

    Name Type Description Default
    css Any

    The CSS content to be commented out. Can be a string, a list of rules, or an object with a .render() method.

    		 required
    return_type Union[str, list]

    Defines the output format. Defaults to an empty string, signifying a string return. If a list instance is passed, it returns a list of commented lines.

    		 str()

    Returns:

    Type Description
    list[str] | str

    Union[str, list]: The commented CSS content. String format: "/ [css] /" List format: ["/ line1 /", "/ line2 /"]
    Example

    css_comment("color: red;") '/ color: red; /'

    css_comment(["color: blue;", "margin: 0;"], return_type=[]) ['/ color: blue; /', '/ margin: 0; /']

    css_style(selectors_rules=None, **declarations)

    A factory function for creating individual CSS rules or bulk Style Bridges.

    This utility serves a dual purpose based on the provided arguments: 1. If selectors_rules is provided: It acts as a factory for a list of SelectorRuleBridge objects, enabling bulk style definitions. 2. If **declarations are provided: It returns a single, validated CssRule object.

    Parameters:

    Name Type Description Default
    selectors_rules dict[CssSelector, CssRule]

    A mapping of selectors to their corresponding rules. Used for bulk component styling.

    		 None
    **declarations dict[str, Any]

    Keyword arguments representing CSS property-value pairs (e.g., color="blue", margin_top="20px").

    		 {}

    Returns:

    Type Description
    str

    Union[List[SelectorRuleBridge], CssRule]: - A list of bridges if selectors_rules is present. - A validated CssRule instance if only **declarations are present.
    Example

    Bulk Pattern

    theme = css_style({CssSelector(".btn"): CssRule(color="red")})

    Quick Rule Pattern

    blue_text = css_style(color="blue", font_weight="bold")

    make_important(css_string)

    Appends the !important flag to CSS declarations.

    This utility modifies a CSS property-value string to ensure it overrides other rules in the cascade. It handles both single declarations and semicolon-terminated strings.

    Parameters:

    Name Type Description Default
    css_string str

    A standard CSS declaration (e.g., "color:red" or "display:block;").

    		 required

    Returns:

    Name Type Description
    str str

    The modified string with !important injected (e.g., "color:red !important;").
    Example
